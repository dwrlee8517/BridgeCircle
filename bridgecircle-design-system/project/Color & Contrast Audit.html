04 · Issue #2 · Critical

Mute2 fails AA — timestamps invisible at small sizes

· Mute2 (#9a8e7d) on Card (#f8f1e2) falls well below the 4.5:1 threshold. It's used for timestamps, metadata, and fine print — typically 10–12px — making them genuinely unreadable for many users. Muted (#7a6e5e) at is borderline, failing for normal-weight text under 18px.

05 · Issue #3 · Moderate

Saffron & Heather buttons fail with white text

White text on Saffron (#b88033) yields :1 — a hard fail. Heather (#8a5e7a) at :1 is borderline. The fix is a single line of logic: when an accent's luminance exceeds 0.18, use Ink (#2a221a) for button text instead of white. Terracotta at :1 also sits just below the AA bar — a slight darkening to #b84e2c solves it cleanly.

Current — all accents with white text
Fixed — smart text color + terracotta darkened
Logic to add to DSButton
jsx
// Luminance-based text color selection
const btnTextColor = (accent) => {
  const lum = getLuminance(accent.hex);
  return lum > 0.18 ? DSC.ink : '#ffffff';
};

// Slightly darken Terracotta for AA compliance

06 · Issue #4 · Enhancement

Semantic tags need a border to read clearly in dense lists

Tag backgrounds at 11–14% opacity are very subtle on the warm card surface. A single 1px border at 30% opacity gives each semantic state an immediate, unambiguous identity — especially important for status chips like "Overdue" or "Waiting 6d" that signal urgent action.

Current — tinted fill only
Proposed — tinted fill + matching 30% border

07 · Visual Identity

Does it look like AI slop?

✓ Genuinely non-generic
  • Warm oat base — not a single gray token in the system
  • Terracotta accent is earthy, specific, and memorable
  • Gradient avatars seeded from person's name — personalised
  • Pill-only shape language applied with strict consistency
  • JetBrains Mono for timestamps is a deliberate, editorial choice
  • 8 named surface treatments — rare depth in a design system
  • No gradient backgrounds, no emoji, no neon, no blue-dominant CTA
  • Density system (compact/comfortable/roomy) is a pro-grade feature
⚠ Things to watch
  • Two intersecting circles as a logo mark is very common — consider a more ownable form
  • Floating pill nav + card grid is a widely used layout — Lifted tone + Inverse spotlight cards will help differentiate
  • Never let a future PR swap terracotta for a blue primary — that's the identity
  • The Lamplight dark tone is only a swatch; if dark mode ships, build it out properly rather than inverting
  • Avoid adding iconography just to fill visual space — your current restraint is an asset
Bottom line

The palette has a distinct, human identity. Fix the four contrast issues and BridgeCircle will be both visually singular and professionally legible — a combination most apps fail to achieve.
