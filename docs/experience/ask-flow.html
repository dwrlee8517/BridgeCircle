BridgeCircle product flow

How an Ask becomes matched people, then a sent request.

The Ask surface owns question-led matching. People remains the broader directory. A typed question stays on /ask, gets translated into matching signals, returns ranked helpers, and carries the original intent into the composer.

2. Target matching pipeline behind Ask results

ADR 0009 upgrades Ask from a single structured candidate engine to hybrid retrieval: hard gates, structured search, lexical search, vector search, warm-network scoring, and LLM reranking.

Gates
Original ask text Kept as the user's wording and passed through the entire flow.
Hard gates first Same org, active member, privacy, helper openness, capacity, not self.
Parallel retrieval
Structured/rules search Explicit filters, helper preferences, school, cohort, city, role fields.
Lexical search Exact names, titles, employers, schools, skills, topics, and profile text.
Vector semantic search Permission-safe profile chunks retrieve helpers with adjacent experience.
Merge and score
Merge evidence Dedupe by user and preserve why each retrieval path found the person.
Warm-network scoring Availability, shared school, cohort, city, relationship path, helper load.
Cap rerank pool Send the strongest 20-40 candidates to the LLM, not the whole directory.
Rerank and fallback
LLM reranks candidates Ranks against the original ask using only privacy-allowed evidence.
Return top 10 Cards use rationale and score bands like "Strong fit" or "Good fit."
If a stage fails Fall back to structured + lexical results, then templated explanations.

Implementation anchors

The Ask UI and send flow exist today. The target matching upgrade is documented in ADR 0009 and is not yet implemented in code.

Ask UI

  • AskBar submits to /ask by default.
  • /ask renders starter state or Ask-specific results.
  • MatchBriefCard carries intent into the CTA URL.

Search backend

  • getMemberSearchResults currently loads auth, org context, counts, and pages.
  • searchAlumniNL is the current baseline: extract filters, search, rerank.
  • ADR 0009 is the target: structured + lexical + vector retrieval, warm scoring, LLM rerank.

Ask creation

  • /ask/new accepts intent and passes it to the form.
  • RequestForm initializes the textarea from the original ask.
  • createAsk writes the row, audit log, notification, and email.
