# BridgeCircle screen templates (Claude Design)

The target-redesign mockups for **every flow and page** in
[`../uploads/FLOWS.md`](../uploads/FLOWS.md), synced down from the `bridgecircle`
Claude Design project. This is the page-and-flow layer that sits beside the
component specimens in [`../preview/`](../preview/). (It superseded the earlier
Codex People/Profile-only `screens/` slice, removed 2026-07-12.)

## What's here

One folder per section. Each `<Name>.dc.html` is a Claude Design **DesignComponent**
(`<x-dc>` template + a `data-dc-script` logic class), rendered by the local
`support.js` (the compiled `dc-runtime`). `ds-base.js` loads
[`../colors_and_type.css`](../colors_and_type.css); `*-data.js` holds sample
content; `app-shell/account-menu.js` is the shared topbar avatar menu.

| Folder | Screens |
|---|---|
| `entry` · `onboarding` | invite verify (+ expired/used), 5-step onboarding (FLOWS §1) |
| `home` | the dashboard (§2) |
| `help` | Help hub, Ask the circle / compose / status / history, Give direct / offer (§3) |
| `people` · `profile` · `profile-self` · `profile-slideover` · `my-circle` | directory, member profile, self profile, Messages slide-over, managed circle (§4, §7) |
| `messages` | one inbox — waiting group, thread, context rail (§5) |
| `school` | events + announcements + newsletter (§6) |
| `notifications` · `settings` | durable bell list, one settings page (§7d) |
| `app-shell` | the desktop sidebar + topbar shell to fill per page |
| `system-states` | not-found / permission-denied / offline / loading (§8) |

## Opening these

They render in the Claude Design environment (the host provides React). Copies
edited **there** are canonical; this tree is a byte-faithful mirror pulled via
DesignSync — re-pull before trusting a local edit. Behavior source is
`../uploads/FLOWS.md`; allowed Toss deviations are in `../uploads/OVERRIDES.md`.

## Not vendored here

- **`fonts/PretendardVariable.woff2`** — the 2 MB self-hosted font exceeds the
  DesignSync pull cap. Every template loads Pretendard from the CDN, so type
  renders correctly; only `../fonts.css`'s `@font-face` dangles. Fetch it from
  the [Pretendard release](https://github.com/orioncactus/pretendard) if a
  self-hosted copy is ever needed.
- Per-template `.thumbnail` previews and `_ds_manifest.json` /
  `_adherence.oxlintrc.json` — app-generated; regenerated by Claude Design.
