# Prototype

Designed screens whose feature is **not yet built** — forward-looking
proposals and work-in-progress explorations. Mirrors the
`product-spec-obsidian-vault/Prototype/` split; a feature may have several
parallel explorations here, and none is authoritative until it ships.

- Name screens after the vault spec they design (e.g. a screen for
  `Prototype/no-invite-landing.md` → `No-Invite Landing.html`).
- A screen **graduates to `Production/`** when its feature lands in mainline —
  move the file in the same change that graduates the spec in the vault.
- Explorations still follow the ledger: token-level divergence from the fork
  belongs in `uploads/OVERRIDES.md` as a *proposed* entry, not inline drift.
