Friction log

where faithful-Toss fought the content — logged, not fixed (feeds the bridgecircle fork)

  1. Mode is invisible below the toggle. Ask and give render as the same surface — white cards on grey-100, blue lead actions, identical topbar. Cover the segmented control and nothing tells you which side you’re on. The system forbids the one move (section-identity color) that would fix it ambiently.
  2. Three identical blue “Offer to help” buttons compete. “One loud action” assumes one action per screen; a browse-list of offers has N equal lead actions, so blue stops meaning “the” action and starts meaning “a” button.
  3. Giving reads transactional. The fill/quiet pair on every row is Toss’s approve/decline dyad — right for payments, cold for “help a neighbor.” Nothing warm is available: green is reserved for status, so generosity gets the same blue as “Send money.”
  4. 30/700 cap undersells the landing moment. “Three people could use you this week” is the emotional headline of the product, and at Display Hero it sits barely larger than a card title. Fine for a utility, small for an invitation.
  5. “Waiting” has no honest hue. The semantic set maps to error/warning/caution/success/info/premium — none fit “pending, that’s okay.” Yellow reads alarming, so Waiting fell back to grey and now looks disabled next to the green “2 offers.”
  6. Avatar tints smuggle meaning. Tinted avatars are the only permitted personality, but the tints come from semantic ramps — a green avatar would read “success,” teal reads “info.” Picking three neutral-but-distinct tints without implying status was guesswork.
  7. Two entry points to the same ask. The Toss idiom wants a fixed bottom CTA, but the text field is already the invitation — “Start an ask” and the placeholder do the same job twice on one screen.
  8. Desktop: the calm turns empty. A 680px column on a 1280px grey-100 canvas leaves most of the screen as blank margin; the idiom has no desktop-scale moves (no wide layouts, no imagery), and the 30/700 cap looks even quieter at this size.
  9. Desktop: the bottom-CTA idiom has no equivalent. The blue action moves inline beside the field, so the mobile and desktop lead actions live in different places with different shapes — parity breaks the moment the system leaves the phone.
