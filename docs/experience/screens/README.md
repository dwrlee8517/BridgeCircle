# Screens

Screen docs bridge product behavior and visual implementation.

Use this folder for decisions that belong to a specific surface, such as layout priority, empty states, loading states, and which design-system patterns a screen should use.

Canonical behavior still lives in specs and architecture. Canonical visual rules still live in [`../ui/design-system/`](../ui/design-system/).
